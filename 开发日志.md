# 开发日志

## 2026-05-24

### 指数对比图修复
- **问题**：`push2his.eastmoney.com` K线接口持续返回 500，导致收益走势页指数对比图无法渲染。客户端 `wx.request` 和云函数 `https.get` 均无法访问该域名（其他东方财富子域名正常）。
- **修复**：`fetchMarketIndex` 云函数主数据源切换为新浪财经 API（`money.finance.sina.com.cn`），东方财富作为备用。新浪使用 `scale=240`（日K线），数据字段映射为 `date/open/close/high/low/volume`。
- **影响**：`cloudfunctions/fetchMarketIndex/index.js` 重写，已部署。

### getPortfolio 云函数性能优化
- 持仓查询从串行 `for` 循环改为 `Promise.all(holdings.map(...))` 并行请求，每个持仓同时发起天天基金 + 东方财富 + NAV历史（可选）请求。
- 新增 `historyDays` 参数，支持一次调用返回 NAV 历史数据（`navHistoryMap`），减少客户端多次调用。
- `fetchNAVHistory` 内置到云函数中，分页拉取（东方财富 `api.fund.eastmoney.com`）。

### 收益走势页性能优化
- 不再对每个持仓单独调 `fetchFundNAVHistory`，改为从 `getPortfolio(80)` 一次性获取所有 NAV 历史。
- 指数数据与 NAV 数据处理并行执行（`indexPromise`）。
- 基金详情页 `checkHolding()` 合并到 `Promise.all` 并行执行。
- 纵轴改为收益率（%），非对比模式不再显示金额。

### 客户端 fetchMarketIndexClient 增强
- 添加重试机制（最多 3 次，指数退避 800ms/1600ms）+ 备用端点（腾讯财经 `web.ifzq.gtimg.cn`）。
- 请求失败时 fallback 到云函数 `fetchMarketIndex`。

### 首页功能
- 持仓超过 8 个支持"加载更多"分页展示。
- 金额全局显示/隐藏切换（仅隐藏金额数字，百分比和净值不隐藏），偏好保存到 Storage。
