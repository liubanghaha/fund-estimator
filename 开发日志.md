# 开发日志

## 2026-05-26

### 基金详情页加载性能优化
- **问题**：详情页首次加载 ~26s+，主要瓶颈在 `fetchFundNAVHistory` 串行获取 260 条记录（13 页 × ~2s/页）。
- **修复**：
  1. **减少初始 NAV 获取量**：默认请求从 260 条降到 80 条（4 页，~8s），覆盖近一月和近三月视图。切换到近半年/近一年时按需获取更多。
  2. **轻量级 checkHolding**：替换 `api.getPortfolio()`（获取全部持仓实时行情）为客户端直查 `holdings` 数据库，用 `fetchEstimate` 已有净值数据计算持仓收益。
  3. **懒加载 profile**：`fetchProfile()` 移出 `fetchAll()`，仅在切换"持仓"或"档案"标签时按需加载。
  4. **云函数参数语义化**：`fetchFundNAVHistory` 参数 `pageSize` → `days`（默认 80），超时 15s → 30s。
- **影响**：首次加载从 ~26s 降至 ~8s（快约 3 倍），消除了 `getPortfolio` 的 2N 个无关 HTTP 请求。
- 云函数新增 `SPX`（标普500）、`IXIC`（纳斯达克）指数支持，数据源为新浪美股实时行情（`gb_inx`、`gb_ixic`）。
- Sina US 格式与 A 股不同（`name, price, change_pct, time, change, open, high, low, ...`），新增 `fetchSinaUSQuote()` 专用解析器。
- **已部署**。

### fetchMarketIndex 云函数重写（实时行情 + 港股多源兜底）
- **问题**：A 股指数显示昨天收盘数据而非实时价格；港股指数（HSTECH、HSI）无数据。
- **修复**：
  - A 股：K 线数据 + 新浪 JS 实时行情（`hq.sinajs.cn`）并行获取后融合，当日数据用实时价格覆盖，新交易日追加数据点。
  - 港股：7 级回退链——东方财富实时 → 新浪实时 → 腾讯实时 → 东方财富全球 K 线 → 新浪港股 K 线 → 腾讯港股 K 线 → Yahoo 兜底。
  - 新增 `fetchEastMoneyRealtime()`、`fetchTencentRealtime()`、`buildQuoteData()`、`formatDate()` 辅助函数。
  - 实时行情接口自动构建 2 日伪 K 线（昨收 + 当前价）。
- **已部署**。

### 港股指数数值全 0 修复
- **问题**：港股指数（HSTECH、HSI）显示了但数值全为 0。
- **根因**：数据源优先级把不可靠的东方财富实时 API 放第一位，且 `fetchTencentRealtime` 硬编码 `prevClose=0`、`open=price`，未解析腾讯 API 的实际字段。
- **验证**： curl 确认新浪（`hq.sinajs.cn`）港股指数无数据（空响应），东方财富（`push2.eastmoney.com`）超时，腾讯（`qt.gtimg.cn`）正常返回完整数据（HSTECH: 4946.88, HSI: 25599.45）。
- **修复**：
  - 港股数据源调整为：腾讯实时 → 东方财富实时 → 新浪实时 → 腾讯 K 线 → 东方财富全球 K 线 → 新浪港股 K 线 → Yahoo。
  - `fetchTencentRealtime` 正确解析 `parts[4]`（昨收）、`parts[5]`（开盘）并传入 `buildQuoteData`。
- **已部署**。

### 首页指数栏收起态间距修复（第二次尝试）

## 2026-05-24

### 指数对比图修复
- **问题**：`push2his.eastmoney.com` K线接口持续返回 500，导致收益走势页指数对比图无法渲染。客户端 `wx.request` 和云函数 `https.get` 均无法访问该域名（其他东方财富子域名正常）。
- **修复**：`fetchMarketIndex` 云函数主数据源切换为新浪财经 API（`money.finance.sina.com.cn`），东方财富作为备用。新浪使用 `scale=240`（日K线），数据字段映射为 `date/open/close/high/low/volume`。
- **影响**：`cloudfunctions/fetchMarketIndex/index.js` 重写，已部署。

### getPortfolio 云函数性能优化
- 持仓查询从串行 `for` 循环改为 `Promise.all(holdings.map(...))` 并行请求，每个持仓同时发起天天基金 + 东方财富 + NAV历史（可选）请求。
- 新增 `historyDays` 参数，支持一次调用返回 NAV 历史数据（`navHistoryMap`），减少客户端多次调用。
- `fetchNAVHistory` 内置到云函数中，分页拉取（东方财富 `api.fund.eastmoney.com`）。

### 收益走势页性能优化
- 不再对每个持仓单独调 `fetchFundNAVHistory`，改为从 `getPortfolio(80)` 一次性获取所有 NAV 历史。
- 指数数据与 NAV 数据处理并行执行（`indexPromise`）。
- 基金详情页 `checkHolding()` 合并到 `Promise.all` 并行执行。
- 纵轴改为收益率（%），非对比模式不再显示金额。

### 客户端 fetchMarketIndexClient 增强
- 添加重试机制（最多 3 次，指数退避 800ms/1600ms）+ 备用端点（腾讯财经 `web.ifzq.gtimg.cn`）。
- 请求失败时 fallback 到云函数 `fetchMarketIndex`。

### 首页功能

- 金额全局显示/隐藏切换（仅隐藏金额数字，百分比和净值不隐藏），偏好保存到 Storage。
