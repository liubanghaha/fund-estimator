## 正文

姐妹们！！我最近参加了 Vibe Coding 的活动，用 AI 写了一个微信小程序，真的被震撼到了 🤯

**我做了什么：**
一个叫「涨跌有数」的基金追踪小程序——截图就能自动识别持仓，实时看收益，还有各种图表分析。从想法到上线，全程对话式开发，一行代码都没手写。

**最让我惊艳的功能：**

📸 **截图导入持仓 — Vibe 感拉满**
截一张支付宝持仓图，AI 自动 OCR 识别基金名称、金额、收益，几秒钟全部入库。百度 OCR + 微信 OCR 双引擎，连表格布局都能智能解析。再也不用一个个手动输入基金代码了，这个体验真的绝！

📊 **收益实时追踪**
交易时段每 5 秒自动刷新收益，首页顶部那个红色渐变的大数字实时跳动。今天的基金是赚是亏，打开就知道，不用再切来切去算了。

📅 **盈亏日历（GitHub 同款热力图）**
每天盈亏变成月历上的彩色小格子，红的赚绿的亏，一年的收益一目了然。这种可视化真的把我看呆了——AI 连 Canvas 2D 绘图都帮我写出来了。

📈 **走势图 + 指数对比**
持仓收益走势叠加沪深 300 / 上证指数对比，还有触摸滑动查看数据点，交互丝滑。基金对比页两边放不同的基金，归一化走势直接对比，再也不纠结选哪只了。

⚡️ **18 个云函数，全是 AI 写的**
后端 18 个云函数——定时快照、OCR 识别、指数拉取、持仓计算——全部通过对话生成。我说"需要一个每小时自动记录收益点的定时任务"，AI 就直接把 trigger 和 handler 写好了。

**开发过程：**
全程用 Claude Code + 微信开发者工具，基本上是：
1. 描述想要的功能
2. AI 写代码 + 调试
3. 我验收
4. 重复

没有碰过微信小程序文档，没有写过 WXML/WXSS，纯 vibe 开发。从最开始的登录页，到 OCR 截图导入，到走势图、日历、基金对比……一个多月迭代到了 v1.1.0。

**感受：**

Vibe Coding 真的不是噱头。对于这种有明确需求、有成熟 API 的产品，AI 完全可以 hold 住全栈开发。关键是——你要清楚自己想要什么，然后准确地描述给 AI。

以前觉得做一个小程序至少要前后端两个人搞两三个月，现在一个人 + AI 几周就搞定了。这种感觉太上头了 🔥
