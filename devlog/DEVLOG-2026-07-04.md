# 开发日志

## 2026-07-04

**开发内容：** 详情页多项优化 + 定投回测修复 + 换手率 + 编辑页加减持仓

**开始时间：** 09:00 | **结束时间：** 10:30

### 功能开发

1. **详情页下拉刷新修复** — scroll-view 吃掉滑动手势导致 `onPullDownRefresh` 无效
   - 改用 scroll-view 内置 `refresher-enabled` + `bindrefresherrefresh`
   - onShow 返回时轻量刷新改为走 `checkHolding()` 查 DB，解决修改持仓后回显不一致

2. **编辑持仓回显数据修正** — `loadHolding` 加载时拉当前净值，用 `份额 × 当前净值` 重算市值和收益，与详情页保持一致

3. **基金换手率展示** — 费用黑洞下方新增「基金换手率」折叠卡片
   - 云函数 `fetchFundProfile` 新增 `fetchTurnoverRate`，从天天基金 HTML 解析换手率表格
   - 展示最近四期（两个半年报 + 两个年报），>200% 标红、<100% 标绿
   - 附说明文字：换手率反映交易频率，费用从净值扣除

4. **定投回测修复**
   - **分页问题**：东方财富 API 固定每页 20 条，之前只拉第 1 页导致大量数据丢失 → 改为按 `TotalCount` 分页并发拉全
   - **年化收益率**：旧公式 `(FV/PV)^(12/months)-1` 是一次性投入算法，严重低估定投收益 → 改用 XIRR 二分法，按每笔实际日期计算

5. **编辑页加减持仓** — `isEdit` 模式下，持有收益下方新增「加减持仓」行
   - `+/−` 切换 + 输入框，光标聚焦时下方展开日期选择器（原生日历）
   - 正数 = 增加份额（加权均价），负数 = 减少份额（校验不超当前份额）
   - 保存时自动拉净值计算、写交易记录（带用户选择日期）、更新持仓
   - 全程无"买/卖/元"字眼，规避微信金融审核

6. **截图识别失败引导**
   - 详情页 / 新增持仓页：识别失败弹窗引导「去搜索」→ 跳转搜索页
   - 加减仓页：去掉原始调试弹窗（暴露引擎名、debug JSON、OCR 原始文本），改为简洁提示
