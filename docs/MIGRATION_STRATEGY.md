# 小程序 → H5 迁移 · 变现 · 运营方案

## 一、广告方案

### 1.1 广告位规划（收入优先 + 体验平衡）

广告仅在**免费用户**展示，Pro 会员全程无广告。

| 广告位 | 位置 | 类型 | 预估 eCPM | 日均展示 | 月收入 |
|--------|------|------|-----------|----------|--------|
| 首页顶部 Banner | 持仓列表上方 | 原生模板 | ¥8-15 | 3,000 | ¥240-450 |
| 自选列表插屏 | 每下滑 5 条 | 原生模板 | ¥8-15 | 2,000 | ¥160-300 |
| 基金详情底部 | Tab 切换时 | Banner | ¥6-12 | 1,500 | ¥90-180 |
| 激励视频 | 查看完整分析前 | 激励视频 | ¥30-60 | 200 | ¥600-1,200 |
| **合计** | | | | | **¥1,090-2,130/月** |

> eCPM 参考微信小程序官方数据：原生模板广告 ¥8-20，激励视频 ¥30-80。
> 保守估计月广告收入 ¥1,000-2,000。

### 1.2 广告位具体设计

#### 1.2.1 首页 Banner

```
┌────────────────────────────┐
│  🌿 今日估算收益  +1383.68 │  ← 资产卡片
│  涨幅 1.23% | 市值 53538   │
├────────────────────────────┤
│  [广告] 某某基金平台        │  ← 原生模板广告（高度自适应）
│  开户送100元券 →            │
├────────────────────────────┤
│  🔍 搜索 | ➕ 新增 | 📝 加减 | ← 快捷操作
│  📊 分析 | ☑️ 批量          │
├────────────────────────────┤
│  持仓列表...               │
```

特性：
- 广告在资产卡片和持仓列表之间，不遮挡核心数据
- 刷新频率：30 秒自动刷新一次
- Pro 会员隐藏此区域

#### 1.2.2 激励视频广告

触发时机：
1. **查看完整走势图**：点击「📈 完整分析」→ 弹出 "观看 15 秒视频，解锁今日深度分析"
2. **数据导出**：点击「导出 CSV」→ 激励视频（Pro 免看）
3. **每日首次打开**：弹窗 "观看视频获取今日Pro体验"

实现：
```js
// 微信小程序激励视频广告
const videoAd = wx.createRewardedVideoAd({ adUnitId: 'xxx' })
videoAd.show().catch(() => {})
videoAd.onClose(res => {
  if (res.isEnded) { /* 发放奖励：解锁Pro功能24h */ }
})
```

收入测算：
- 日均 200 次观看 × ¥0.50/次 = ¥100/天 = ¥3,000/月
- 这是广告收入的主要来源

#### 1.2.3 不展示广告的场景

- Pro 会员全程无广告
- 支付中/加载中不展示
- 用户操作后 5 秒内不弹广告
- 首次使用前 3 天不展示广告

### 1.3 广告合规注意事项

| 规则 | 做法 |
|------|------|
| 禁止遮挡功能 | 广告不覆盖按钮/输入框 |
| 禁止诱导点击 | 不使用"点一下""帮忙点"等诱导文案 |
| 禁止自动播放音频 | 仅使用静音视频 |
| 禁止金融广告 | 广告单元设置中屏蔽金融/股票/基金类广告 |
| 激励视频必须明示 | "观看视频解锁XX" 文案清晰 |

### 2.5 模板消息体系

微信小程序订阅消息，参考微信官方规范：

| 消息场景 | 模板 | 推送时间 | 触发条件 |
|----------|------|----------|----------|
| 每日收益 | 收益通知 | 15:30 | 交易日自动推送 |
| 止盈提醒 | 涨跌提醒 | 实时 | 涨幅超阈值 |
| 止损提醒 | 涨跌提醒 | 实时 | 跌幅超阈值 |
| PE 变化 | 自定义提醒 | 实时 | PE 档位变化 |
| 周报 | 数据报告 | 周五 17:00 | 用户订阅 |
| 系统通知 | 服务通知 | 随时 | 审核/更新等 |

> ⚠️ 注意：微信已限制订阅消息只能用于"用户主动订阅的场景"
> 策略：在用户完成某个操作后，自然引导"开启通知"
> 例：添加持仓后 → "开启收益通知，每天收盘自动推送"

## 四、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 小程序审核再被拒 | 入口切断 | 提前建立微信群，直接引流 H5 |
| 用户不愿去 H5 | 流失 | 核心功能（持仓/收益）保留在小程序 |
| H5 支付无法实现 | 变现受阻 | 走小程序支付，云端同步会员 |
| 竞品出现 | 用户分流 | 强化社群 + 数据优势（持仓历史） |
| 广告影响体验 | 用户流失 | 控制频率，Pro 会员无广告 |
